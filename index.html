1. Setup & Configuration

Initial Setup

  1. Install Git: https://git-scm.com/downloads
  2. Configure user: 
    git config --global user.name "Your Name"
git config --global user.email "your@email.com"
  3. Clone team repository: git clone https://github.com/team/repo.git

3. Branching Strategy

Effective Branch Management

  • main - Production-ready code
  • develop - Integration branch
  • feature/ - New functionality
  • hotfix/ - Critical bug fixes
  • release/ - Preparation for deployment

Creating a Feature Branch:

git checkout -b feature/user-authentication

4. Conflict Resolution

Resolving Merge Conflicts

  1. Fetch latest changes: git fetch origin
  2. Rebase your branch: git rebase origin/main
  3. Locate conflict markers in files: 
    <<<<<<< HEAD
Your changes
=======
Incoming changes
>>>>>>>
  4. Edit files to keep correct changes
  5. Mark as resolved: git add .
  6. Continue rebase: git rebase --continue

5. GitHub Collaboration

Pull Request Lifecycle

  1. Push your branch: git push origin feature-branch
  2. Create PR on GitHub repository
  3. Add description and reviewers
  4. Address review comments:
    • Make local changes
    • Commit with git commit --amend
    • Force push: git push -f
  5. Merge after approval

Code Review Tips

  • Check for clear naming conventions
  • Verify error handling
  • Ensure consistency with project patterns
  • Confirm test coverage

6. Essential Command Reference

Command Description When to Use
git status Show current branch status Before/after any operation
git log --oneline --graph Visualize branch history Understanding project history
git diff Show uncommitted changes Before committing
git stash Temporarily save changes Switching branches with uncommitted work
git cherry-pick [commit] Apply specific commit Porting hotfixes between branches

Common Issues & Solutions

Accidental commit to main branch?
  1. Create new branch: git branch fixes
  2. Reset main: git reset --hard origin/main
  3. Checkout fixes branch: git checkout fixes
Lost local changes?

Check reflog: git reflog to find lost commits
